# Техническое задание
## На разработку программного обеспечения "Служба знакомств"
### 1. Общие сведения
**Полное наименование системы:** Программное обеспечение службы знакомств с комплексом подбора кандидатов по заданным критериям.
**Разработчик:** Хисамиев Владимир Игоревич
### 2. Назначение и цели создания
#### 2.1 Назначение
Автоматизация процессов поиска и подбора кандидатов для общения и знакомства.
#### 2.2 Цели разработки
- Разработка клиентского интерфейса на C# Windows Forms
- Реализация логики авторизации и регистрации пользователей
- Создание и отображение пользовательских анкет
- Реализация механизма чата между пользователями
- Подключение к базе данных MySQL
- Создание архитектурной основы для последующей реализации фильтрации
### 3. Требования к системе
#### 3.1 Функциональные требования
**Регистрация и авторизация:**
- Проверка уникальности логина
- Валидация введенных данных
- Хеширование паролей
**Управление анкетой:**
- Заполнение личной информации
- Добавление интересов
- Загрузка фотографий
**Просмотр анкет:**
- Отображение списка пользователей
- Фильтрация по критериям (возраст, пол, город)
**Система лайков:**
- Возможность отметить понравившегося пользователя
- Определение взаимных симпатий
**Чат:**
- Обмен сообщениями между пользователями
- История переписки
- Список активных диалогов
**Управление аккаунтом:**
- Редактирование профиля
- Смена пароля
- Выход из системы
#### 3.2 Нефункциональные требования
**Производительность:**
- Время отклика не более 2 секунд
- Оптимизация SQL-запросов
**Безопасность:**
- Хеширование паролей (SHA256)
- Защита от SQL-инъекций
- Валидация входных данных
**Надежность:**
- Обработка исключительных ситуаций
- Сохранение целостности данных
**Удобство использования:**
- Интуитивно понятный интерфейс
- Навигация по вкладкам
### 4. Требования к программному обеспечению
#### 4.1 Системное ПО
- Операционная система: Windows 10/11
- Платформа: .NET Framework / .NET 6.0
- СУБД: MySQL 8.0
#### 4.2 Технологии разработки
- Язык программирования: C#
- Технология GUI: Windows Forms
- Доступ к данным: ADO.NET + MySQL Connector
### 5. Требования к структуре и функционированию
#### 5.1 Состав системы
- Модуль авторизации (LoginForm)
- Главное окно (MainForm)
- Модуль просмотра анкет (ProfilesControl)
- Модуль чата (ChatControl, ChatWindowControl)
- Модуль управления аккаунтом (AccountControl)
- Модуль работы с БД (DbConfig)
#### 5.2 Структура базы данных
**Таблицы:**
- `users` — учетные записи пользователей
- `likes` — симпатии между пользователями
- `chats` — диалоги
- `messages` — сообщения
#### 5.3 Схема взаимодействия компонентов
LoginForm → MainForm → [ProfilesControl, ChatControl, AccountControl] → DbConfig → MySQL
